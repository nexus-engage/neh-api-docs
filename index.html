NEH API Documentation

The NEH API provides programmatic access to visit management, account/customer data, person records, opportunities, surveys, and supporting reference data. All endpoints are under /api/v3/ and return JSON. It is designed for CRM integrations that need to create visit requests, sync account and person data, manage sales opportunities, and retrieve survey responses.

Authentication

All API requests require OAuth 2.0 authorization

OAuth 2.0 — Client Credentials flow

1
Obtain credentials
Request a client_id and client_secret from the NEH platform administrator for your environment.
2
Request an access token
POST your credentials to the token endpoint. The token URL is provided per-environment (see your environment configuration for tokenUrl).
3
Include the token on every request
Pass the token in the Authorization header with the format: Bearer {token}

Error responses

401Missing or invalid token. Re-authenticate and retry.
403Token valid but insufficient permissions for this resource.
